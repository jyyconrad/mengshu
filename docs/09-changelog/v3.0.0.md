# memory-autodb v3.0 Changelog

## [3.0.0] - 2026-06-08

### 🎉 重大升级：5 问题语义协议 + Agent 快路径

这是 memory-autodb 的首次架构级升级，引入了**5 问题语义协议**和**Agent 快路径**，大幅提升 Agent 启动性能和记忆召回质量。

---

### ✨ 新增功能

#### 5 问题语义协议（Milestone 1-2）
- **MemorySemanticType**: 5 个语义类型对应 Agent 执行前必须知道的问题
  - `profile`: Q1 我为谁工作？（输出风格、风险偏好）
  - `task_context`: Q2 我在做什么？（当前目标、优先级）
  - `rules`: Q3 什么不能做？（行为边界、合规底线）
  - `experience`: Q4 之前怎么做过？（历史经验、决策依据）
  - `resource`: Q5 有什么可用资源？（工具、文档、连接器）

- **可选字段**: `semanticType` 作为 `MemoryRecord` 的可选字段，不强制所有记忆归类
- **向后兼容**: 保留 `kind` 字段，无法归类的记忆仍可通过 `memory_lookup` 检索

#### Agent 快路径（Milestone 1）
- **4 个时点接口**:
  1. `memory_context_fast`: 任务启动，获取 5 槽位上下文（P95 < 80ms）
  2. `observe_light`: 运行中轻量提交 observation
  3. `lookup`: 任务中按需速查
  4. `session_commit`: 会话结束，入队后台 job

- **REST API**: 
  - `POST /v1/agent/context`
  - `POST /v1/agent/observe`
  - `POST /v1/agent/lookup`
  - `POST /v1/agent/session/commit`

- **OpenClaw MCP Tool**: `memory_context_fast`（与现有工具共存）

#### 候选区与治理（Milestone 2）
- **CandidateRecord**: 自动抽取的记忆先进入候选区，待审核后升格为正式记忆
- **状态机**: `pending` → `approved` / `rejected` / `archived` / `expired`
- **自动淘汰**: 30 天未命中 → 删除；命中过未确认 → 归档
- **批量审核**: approve / reject / archive / by_filter / evict_expired
- **5 Type Extractor**: 启发式抽取器（无 LLM 依赖），基于关键词正则匹配

#### Slot Context Builder
- **SlotSnapshot 缓存**: 按 `scope + semanticType` 分组，支持差异化 TTL
  - profile: 30min / task_context: 5min / rules: 60min / experience: 15min / resource: 10min
- **预算控制**: latency budget / token budget / 降级策略
- **Prompt-safe 输出**: `<relevant-memories>` 格式，自动转义注入风险

#### kind → semanticType 映射
- **v0.1 高置信度映射**（覆盖率 ~40%）:
  - `goal` → `task_context`
  - `document` → `resource`
  - `knowledge` → `resource`
- **unmappable 策略**: `fact` / `entity` / `observation` / `other` 保留为 kind-only

#### 新增类型
- `MemorySemanticType`: 5 问题语义类型
- `MemoryContainer`: 记忆容器（personal / project / session_candidate / team / enterprise）
- `MemoryLifecycleStatus`: 生命周期状态（active / archived / revoked / superseded / promoted）
- `MemoryVisibility`: 可见性（private / workspace / team / public）
- `MemoryEdge`: 工作记忆图谱关系（v0.4+）
- `GraphNode`: 统一图谱节点（v0.4+）

### 📊 性能优化

- **SlotSnapshot 缓存**: 避免每次请求都重新聚合记忆
- **差异化 TTL**: 按记忆类型设置不同缓存时长
- **importance × hotness 排序**: 优先召回高重要性且被频繁命中的记忆
- **预算截断**: latency budget / token budget，超预算时返回降级结果

### 💡 使用示例

#### 使用 memory_context_fast（OpenClaw）
```typescript
// Agent 启动时调用
const result = await memory_context_fast({
  task: "完成 memory-autodb 架构升级",
  tokenBudget: 4000,
  latencyBudgetMs: 80,
});

// 输出包含 5 槽位上下文
console.log(result.slots.rules);      // Q3: 什么不能做？
console.log(result.slots.task_context); // Q2: 我在做什么？
console.log(result.telemetry);        // { latencyMs, nodesUsed, cacheHit }
```

#### 使用 REST API
```bash
curl -X POST http://127.0.0.1:3847/v1/agent/context \
  -H "Content-Type: application/json" \
  -d '{
    "task": "完成架构升级",
    "tokenBudget": 4000,
    "latencyBudgetMs": 80
  }'
```

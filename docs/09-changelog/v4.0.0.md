# v4.0.0 Memory Middleware Changelog

## 新增

- 抽出 `MemoryService`、`MemoryScope`、legacy mapping 和 OpenClaw adapter。
- 新增本机 REST server、JS SDK、MCP tools adapter、`ltm serve/status/health`。
- `memory_scan_directory` / `ltm scan` 改走 deterministic ingestion pipeline，返回 jobs/chunks 统计。
- 新增 repository contracts、in-memory jobs/audit/documents/chunks baseline。
- 新增 BM25 text index、RRF fusion、retrieval orchestrator、Context Packer。
- 新增结构化 graph：entity/relation 类型、规则抽取、graph query、`/v1/graph/query`。
- 新增 Memory Tree：source buffer/seal、topic hotness routing、global daily digest。
- 新增 Web Console API 和 `/console` 静态界面，支持 overview、quick lookup、graph、jobs。
- 新增 v4 migration dry-run、lifecycle audit 和 retention sweep helper。

## 兼容性

- 保持 OpenClaw `memory_store`、`memory_recall`、`memory_scan_directory`、`memory_cleanup` 工具兼容。
- 保持旧 `memories` / `knowledge` 表和 legacy provider adapter 可用。
- v4 新 schema 当前以 contract baseline 和 dry-run 方式推进，不强制迁移旧数据。

## 核心变更

### 全局配置目录升级

**背景**：memory-autodb 从"OpenClaw 的长期记忆插件"升级为"面向 Agent 应用的本地优先记忆中间件"，全局文件不应继续挂在 `~/.openclaw/` 下，否则会带来产品边界不清、迁移和备份困难、跨产品 Working Context 不自然等问题。

**新全局目录**：`~/.memory-autodb/`（替代 `~/.openclaw/`）

**目录结构**：
```
~/.memory-autodb/
  config.json                 # 全局运行配置，不直接保存密钥明文
  .env                        # 本机密钥和敏感环境变量
  registry.json               # 本机 workspace/project registry 快速索引
  memory/
    lancedb/                  # 默认本地 LanceDB 数据库
  projects/
    <projectId>/
      manifest.json           # 项目完整 manifest，长期 identity 真源
      evidence/               # 原始证据指针、canonical source 或快照
      indexes/                # 可重建索引状态、manifest diff、content hash
      exports/                # 可读导出、备份、迁移包
      tree/                   # source/topic/global tree 的持久状态
      audit/                  # 项目级审计、commit、refresh 记录
  logs/
  backups/
```

**配置/密钥分离**：
- `~/.memory-autodb/config.json`：全局运行策略，可备份，不保存密钥明文
- `~/.memory-autodb/.env`：本机密钥（权限 `0600`），不纳入版本管理
- 所有密钥引用采用 `${...}` 占位符，运行时从 `.env` 解析

**项目指针化**：
- **项目目录**：`.memory-autodb.json` 轻量指针（version: "0.2"）
  ```json
  {
    "version": "0.2",
    "projectId": "proj-xxxx",
    "workspaceId": "ws-xxxx",
    "globalManifest": "~/.memory-autodb/projects/proj-xxxx/manifest.json",
    "createdAt": 1780000000000
  }
  ```
- **全局目录**：`~/.memory-autodb/projects/<projectId>/manifest.json` 完整状态
  - 包含 `displayName`、`rootHints`、`scopeDefaults`、`slotReusePolicy`、`sourceRoots` 等完整配置
  - 长期 identity 真源，支持项目目录移动后重新绑定

**本机项目索引**：`~/.memory-autodb/registry.json`
- 快速列出本机项目
- 支持项目目录移动后的重新绑定
- 支持 Web Console 的整体预览

**兼容性**：旧 `~/.openclaw/` 路径仍可读取，输出 warning 提示迁移

### 统一客户端接入

所有 MCP 客户端统一指向同一启动脚本、同一配置文件和同一 `.env`。

**环境变量**：
- `MEMORY_AUTODB_HOME`：覆盖全局目录（默认 `~/.memory-autodb`）
- `MEMORY_AUTODB_CONFIG`：覆盖配置文件路径（默认 `~/.memory-autodb/config.json`）
- `MEMORY_AUTODB_ENV`：覆盖 .env 文件路径（默认 `~/.memory-autodb/.env`）

**Codex 接入**（`pyproject.toml`）：
```toml
[mcp_servers.memory_autodb]
command = "/path/to/memory-autodb/node_modules/.bin/tsx"
args = ["/path/to/memory-autodb/scripts/memory-autodb-mcp.ts"]

[mcp_servers.memory_autodb.env]
MEMORY_AUTODB_CONFIG = "/Users/<user>/.memory-autodb/config.json"
MEMORY_AUTODB_ENV = "/Users/<user>/.memory-autodb/.env"
NO_PROXY = "127.0.0.1,localhost,::1"
no_proxy = "127.0.0.1,localhost,::1"
```

**Claude Code / Claude Desktop 接入**（`.claude/mcp.json`）：
```json
{
  "mcpServers": {
    "memory-autodb": {
      "command": "/path/to/memory-autodb/node_modules/.bin/tsx",
      "args": ["/path/to/memory-autodb/scripts/memory-autodb-mcp.ts"],
      "env": {
        "MEMORY_AUTODB_CONFIG": "/Users/<user>/.memory-autodb/config.json",
        "MEMORY_AUTODB_ENV": "/Users/<user>/.memory-autodb/.env",
        "NO_PROXY": "127.0.0.1,localhost,::1",
        "no_proxy": "127.0.0.1,localhost,::1"
      }
    }
  }
}
```

**OpenClaw 接入**（`openclaw.plugin.json`）：
```json
{
  "slots": {
    "memory": "memory-autodb"
  },
  "entries": {
    "memory-autodb": {
      "enabled": true,
      "configPath": "~/.memory-autodb/config.json",
      "envPath": "~/.memory-autodb/.env"
    }
  }
}
```

### 新增命令

#### `ltm migrate-home`

一键迁移 `~/.openclaw/` → `~/.memory-autodb/`。

**特性**：
- **默认 dry-run 模式**：不带 `--execute` 时只打印计划，不执行
- **备份机制**：`--backup` 选项时，迁移前先备份到 `~/.openclaw.backup-<timestamp>/`
- **冲突检测**：目标文件已存在时询问是否覆盖（默认跳过）；`--force` 时直接覆盖
- **迁移后验证**：校验关键文件存在性（.env、config.json、memory/lancedb/）
- **registry 更新提示**：扫描项目指针，提示用户可能需要重新 `ltm init`

**用法**：
```bash
# 预览迁移计划（不执行）
ltm migrate-home

# 执行迁移并备份
ltm migrate-home --execute --backup

# 执行迁移并强制覆盖冲突文件
ltm migrate-home --execute --force
```

**迁移清单**：
1. `~/.openclaw/.env` → `~/.memory-autodb/.env`
2. `~/.openclaw/memory-autodb-mcp.json` → `~/.memory-autodb/config.json`
3. `~/.openclaw/memory/` → `~/.memory-autodb/memory/`（递归复制）
4. `~/.openclaw/conf/plugins.json` 中的内联配置 → 提示手工迁移

### 变更的命令

#### `ltm init`

**新行为**（v0.1.2）：
1. 写项目指针（`.memory-autodb.json` version: "0.2"）
2. 写全局 manifest（`~/.memory-autodb/projects/<projectId>/manifest.json`）
3. 自动注册到 `registry.json`
4. **幂等**：重复 init 更新 registry 的 `lastOpenedAt`，不修改已存在的 `projectId`

**旧行为**（v0.1.1）：
- 只写项目指针，不创建全局项目目录
- 不更新 registry

## 升级指南

### 新安装

默认使用 `~/.memory-autodb/`，无需额外配置。

### 已有用户

#### 方式一：自动迁移（推荐）

```bash
# 预览迁移计划
ltm migrate-home

# 执行迁移（带备份）
ltm migrate-home --execute --backup

# 验证
ltm stats
```

#### 方式二：手动迁移

1. 复制配置文件：
   ```bash
   mkdir -p ~/.memory-autodb
   cp ~/.openclaw/memory-autodb-mcp.json ~/.memory-autodb/config.json
   cp ~/.openclaw/.env ~/.memory-autodb/.env
   ```

2. 复制数据目录：
   ```bash
   cp -r ~/.openclaw/memory ~/.memory-autodb/
   ```

3. 更新已有项目（可选）：
   ```bash
   cd /path/to/your/project
   ltm init --force
   ```

#### 方式三：保留旧路径

如无需迁移，旧路径仍可用（会输出 warning）。设置环境变量显式指定：
```bash
export MEMORY_AUTODB_HOME=~/.openclaw
```

### 迁移后步骤

1. 验证配置：`ltm stats`
2. 测试功能：`ltm search "test"`
3. 更新客户端配置（Codex/Claude Desktop/OpenClaw）指向新路径
4. 确认无问题后可删除旧目录：`rm -rf ~/.openclaw`

## 技术架构

### 模块化路径解析

**新增模块**：`core/paths.ts`

统一管理所有全局目录相关路径：
- `resolveHomeDir`：解析全局目录（支持 `MEMORY_AUTODB_HOME` 覆盖）
- `resolveConfigPath`：全局 config.json 路径
- `resolveEnvPath`：全局 .env 路径
- `resolveRegistryPath`：全局 registry.json 路径
- `resolveProjectDir`：项目目录路径
- `resolveProjectManifestPath`：项目 manifest.json 路径
- `expandHome`：展开 `~` 为绝对路径

**设计原则**：
- 只解析路径字符串，不读文件、不创建目录、不输出日志
- 路径相关的兼容回退（如旧 `~/.openclaw/`）放在调用方
- 不依赖 zod/任何第三方解析库，确保启动早期可调用

### 本机项目索引

**新增模块**：`core/registry.ts`

维护全局 `registry.json`，提供：
- `readRegistry`：读取 registry（不存在时返回空结构）
- `writeRegistry`：原子写入 registry（tmp + rename）
- `upsertProject`：插入或更新项目记录
- `touchProjectOpenedAt`：更新项目最后打开时间
- `removeProject`：从 registry 移除项目

**registry 格式**：
```json
{
  "version": 1,
  "projects": {
    "proj-xxxx": {
      "workspaceId": "ws-xxxx",
      "displayName": "memory-autodb",
      "manifestPath": "~/.memory-autodb/projects/proj-xxxx/manifest.json",
      "lastSeenRoot": "/Users/user/develop/code/work_code/openclaw_plugins/memory-autodb",
      "lastOpenedAt": 1780000000000
    }
  },
  "workspaces": {
    "ws-xxxx": {
      "projectIds": ["proj-xxxx"]
    }
  }
}
```

### 项目指针化

**变更模块**：`adapters/openclaw/manifest.ts`

**v0.2 指针格式**：
```json
{
  "version": "0.2",
  "projectId": "proj-xxxx",
  "workspaceId": "ws-xxxx",
  "globalManifest": "~/.memory-autodb/projects/proj-xxxx/manifest.json",
  "createdAt": 1780000000000
}
```

**v0.1 完整格式**（已弃用）：
```json
{
  "version": "0.1",
  "projectId": "proj-xxxx",
  "workspaceId": "ws-xxxx",
  "displayName": "project-name",
  "scopeDefaults": { ... },
  "slotReusePolicy": { ... },
  ...
}
```

**兼容性**：支持从 v0.1 自动升级到 v0.2（创建全局 manifest 并回填指针）

## 附录：兼容策略

### 配置文件优先级

1. 如果显式传入 `MEMORY_AUTODB_CONFIG`，优先使用它
2. 否则读取 `~/.memory-autodb/config.json`
3. 若不存在，再兼容读取 `~/.openclaw/memory-autodb-mcp.json` 或 `~/.openclaw/conf/plugins.json`
4. 读取旧路径时输出 warning，提示用户迁移

### 项目 manifest 兼容

1. v0.1 `.memory-autodb.json` 直接包含较多字段；v0.2 指针化后支持自动升级
2. 如果项目目录只有旧 `.memory-autodb.json`，`ltm init` 不覆盖 identity，而是创建全局项目目录并回填指针
3. 如果 `~/.memory-autodb/projects/<projectId>/manifest.json` 缺失，允许从项目指针恢复，但标记为 degraded

### 旧路径支持周期

- **v0.1.2**：兼容读取，输出 warning
- **v0.2.0**：兼容读取，输出 deprecation warning
- **v0.3.0**：移除旧路径支持（只保留显式 `MEMORY_AUTODB_HOME` 覆盖）

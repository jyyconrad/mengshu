# v2.1.0 - 2026-03-11

## 新增

### 多存储分类架构

- **用户友好的存储分类名称**
  - `核心记忆`：存储对话记忆（用户偏好、决策、实体信息等）
  - `知识库`：存储文档知识（扫描的 Markdown 文件、技术文档等）
  - 支持跨分类搜索和隔离查询

- **存储分类映射**
  - `核心记忆` → `memories` 表
  - `知识库` → `knowledge` 表

### 增强型 CLI 命令

- `ltm tables` - 列出所有可用的存储分类
- `ltm query` - 高级 JSON 过滤查询
- `ltm export` - 导出数据为 JSON/CSV 格式
- 现有命令增强：`ltm search`、`ltm stats`、`ltm cleanup` 支持分类参数

### 元数据自动丰富

- 自动捕获 OpenClaw 上下文:
  - `sessionId`、`conversationId`、`messageId`、`userId`
  - `projectPath`、`workspacePath`
  - `agentId`、`agentName`
  - `groupId`、`groupName`
  - `userName`、`userEmail`
- 技术元数据:
  - `embeddingModel`、`pluginVersion`、`language`、`source`
- 文档元数据（扫描时）:
  - `filePath`、`fileModifiedAt`、`directoryPath`、`tokenCount`

### 配置选项扩展

- `scanner.targetTable` - 指定扫描目标存储分类
- `scanner.autoEnrichMetadata` - 自动丰富元数据
- `tables` - 表级配置（启用/自动索引）

## 改进

### 批量处理优化

- 批量向量化：每批最多 20 个文本
- 并发控制：最多 3 个并发请求
- 自动重试：最多 3 次重试

### 错误处理增强

- Prompt 注入检测
- 内容自动转义
- 更清晰的错误信息

## 修复

- 无

## 变更

### 数据库 Schema

- `memories` 表：新增 `content_hash` 唯一索引
- `knowledge` 表：新增 `content_hash` 唯一索引
- 元数据字段扩展为 JSONB 类型

### API 变更

- `memory_store` 新增 `storageCategory` 参数
- `memory_recall` 新增 `category`、`searchAll` 参数
- `memory_scan_directory` 新增 `category`、`autoEnrichMetadata` 参数

# SR-001 - 内存插件功能需求

## 关联业务需求

- BR-001: 长期记忆存储需求

## 功能需求

### FR-1 记忆存储

**FR-1.1 手动存储**
- 提供 `memory_store` 工具供用户调用
- 支持指定重要性、分类、元数据
- 自动去重（基于内容哈希）

**FR-1.2 自动捕获**
- 在 `agent_end` 钩子中触发
- 识别对话中的重要信息
- 自动提取并存储

**FR-1.3 文档扫描**
- 提供 `memory_scan_directory` 工具
- 支持 Markdown 文件解析
- 文本切分和向量化
- 批量处理和并发控制

### FR-2 记忆检索

**FR-2.1 向量搜索**
- 提供 `memory_recall` 工具
- 支持语义相似度匹配
- 可配置返回数量

**FR-2.2 自动召回**
- 在 `before_agent_start` 钩子中触发
- 根据用户消息自动检索相关记忆
- 注入到上下文

**FR-2.3 高级查询**
- CLI `query` 命令支持 JSON 过滤
- 支持跨表搜索
- 支持元数据过滤

### FR-3 数据管理

**FR-3.1 统计**
- CLI `stats` 命令显示统计信息
- 按类型、分类统计

**FR-3.2 清理**
- CLI `cleanup` 命令清理旧数据
- 支持按时间、过滤器清理

**FR-3.3 导出**
- CLI `export` 命令导出数据
- 支持 JSON、CSV 格式

### FR-4 存储架构

**FR-4.1 多表存储**
- `memories` 表：对话记忆
- `knowledge` 表：文档知识

**FR-4.2 存储分类**
- 核心记忆（memories 表）
- 知识库（knowledge 表）

**FR-4.3 混合模式**
- LanceDB：本地向量索引
- Supabase：云端持久化

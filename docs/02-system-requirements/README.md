# 系统需求

本目录把业务需求转成系统功能、非功能约束和验收口径。这里描述“系统必须提供什么能力”，不展开模块内部实现。

## 维护规则

- 新系统需求使用 `SR-{序号}-{主题}.md` 命名。
- 每篇需求必须链接关联业务需求。
- 改 API、CLI、schema 或测试验收时，同时检查 `../05-api/`、`../06-database/`、`../07-test/`。

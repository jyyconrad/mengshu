# 01-business-requirements 业务需求

## 用途
记录业务方提出的原始需求，描述"做什么"和"为什么做"

## 文件命名
`BR-{序号}-{标题}.md`

## 编号规则
`BR-001`, `BR-002`, `BR-003`... (按创建顺序)

## 文件模板
```markdown
# BR-{序号} - {标题}

## 背景
{描述业务背景和痛点}

## 目标
{描述期望达成的目标}

## 需求描述
{详细描述需求内容}

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 优先级
{P0/P1/P2}

## 创建信息
- 创建人：{姓名}
- 创建日期：{YYYY-MM-DD}
- 最后更新：{YYYY-MM-DD}
```

## 示例
```
01-business-requirements/
├── BR-001-支持 Python 代码扫描.md
├── BR-002-支持前端代码扫描.md
└── BR-003-敏感信息检测需求.md
```

## 维护方式
用户描述需求，AI 整理成文档

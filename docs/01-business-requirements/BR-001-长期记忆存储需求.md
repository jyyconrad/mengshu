# BR-001 - 长期记忆存储需求

## 背景

AI 助手在会话过程中会产生大量有价值的信息，包括：
- 用户偏好和习惯
- 重要决策和共识
- 项目知识点
- 技术文档和笔记

这些信息在会话结束后容易丢失，用户需要一种方式将其持久化存储，以便后续检索和复用。

## 目标

1. **持久化存储**：将对话中的重要信息存储到向量数据库，支持语义搜索
2. **自动捕获**：智能识别对话中的关键信息，自动存储
3. **自动召回**：在后续对话中自动检索相关记忆，提供上下文
4. **知识库构建**：支持扫描目录构建本地知识库
5. **混合存储**：支持本地存储和云端存储，满足不同场景需求

## 需求描述

### 功能需求

1. **记忆存储**
   - 支持手动存储指定内容
   - 支持自动捕获对话内容
   - 支持批量导入文档

2. **记忆检索**
   - 支持向量相似度搜索
   - 支持元数据过滤
   - 支持多表/多分类查询

3. **数据管理**
   - 支持数据清理
   - 支持数据导出
   - 支持数据统计

4. **存储模式**
   - 本地 LanceDB 存储
   - 云端 Supabase 存储
   - 混合模式（向量本地 + 数据云端）

### 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 搜索响应时间 < 1 秒 |
| 扩展性 | 支持 10 万 + 记忆条目 |
| 可靠性 | 数据持久化，不丢失 |
| 易用性 | 提供 CLI 和工具调用两种使用方式 |

## 验收标准

- [ ] 能够存储和检索文本记忆
- [ ] 支持语义相似度搜索
- [ ] 支持批量文档扫描
- [ ] 支持本地和云端存储
- [ ] 提供 CLI 管理命令
- [ ] 自动捕获和召回功能正常工作

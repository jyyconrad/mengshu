# update-doc

**用途**：统一文档管理入口
**输出**：当前项目目录的 `./docs/`

---

## 三种模式

### 初始化模式

触发词：
- "初始化文档系统"
- "建立文档规范"
- "新项目需要文档管理"

执行：创建 `./docs/` 完整目录结构（9 个子目录 + 各目录 README）、`.claude/` 本地配置、可选 Hooks。

### 更新模式

触发场景及对应文档：

| 用户说 | 创建/更新文档 |
|--------|--------------|
| "需要支持 XX 功能" | 业务需求 + 系统需求 |
| "添加 XX API" | API 文档（+ 变更日志） |
| "数据库 Schema 变了" | 数据库设计文档 |
| "XX 接口报错 / 发现 Bug" | 缺陷记录 + 变更日志 |
| "准备发布 v1.x" | 变更日志 |
| "设计 XX 模块" | 模块设计（04.1）或详细设计（04.2） |

### 健康检查模式

触发词：
- `/doc-health`
- "检查文档状态"
- "文档健康度"

执行：运行健康检查脚本，输出评分报告和改进建议。

---

## 执行流程

```
意图识别 → 判断复杂度 → [复杂时] 出计划等确认 → 执行 → 告知结果
```

**何时先出计划**：
- 初始化（创建 9 个目录 + 多文件）
- 单次涉及 3 个以上文档的更新

**何时直接执行**：
- 单文档更新（如只新增一个 API doc）
- 健康检查

---

## 关联命令

当更新涉及**模块新增或架构调整**时，建议同步执行：
- `/update-codemaps` — 更新代码结构索引（给 AI 用的代码导航地图）

---

**记住**: 文档与代码同等重要，保持同步是团队协作的基础。

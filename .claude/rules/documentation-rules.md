# Documentation Rules

> **用途**: 项目文档管理规范（原则层）
> **范围**: 全局适用
> **更新**: 2026-03-11

---

## 标准文档目录结构

所有项目采用统一的文档目录结构：

```
docs/
├── 01-business-requirements/    # 业务需求 - 描述"做什么"和"为什么"
├── 02-system-requirements/      # 系统需求 - 功能点清单和非功能约束
├── 03-architecture/             # 架构设计 - 技术架构、模块划分、技术选型
├── 04-design/
│   ├── 04.1-overview/          # 模块设计 - 模块边界、交互关系、错误处理策略
│   └── 04.2-detail/            # 详细设计 - 具体实现方案、算法、边界条件
├── 05-api/                      # API 文档 - REST 接口定义（含 Headers、Body、示例）
├── 06-database/                 # 数据库设计 - 表结构、字段说明、索引
├── 07-test/                     # 测试用例 - 功能测试、边界测试
├── 08-defects/                  # 缺陷记录 - 根因分析、修复记录
└── 09-changelog/                # 变更日志 - 版本追踪
```

---

## 核心原则

### 1. 文档即代码

- 文档与代码同等重要
- 文档与代码同步更新
- 文档需要版本控制

### 2. 单一事实来源

- 从代码生成文档（API、数据库）
- 避免手动维护可自动生成的内容
- 文档必须反映代码实际状态

### 3. 增量更新

- 每次只更新必要的文档
- 避免一次性更新所有文档
- 保持文档简洁

### 4. 一致性

- 文档与代码一致
- 各文档之间一致
- 使用标准模板

### 5. 版本追踪

- 每次变更都要记录
- 使用语义化版本号（major.minor.patch）
- 记录贡献者和关联 PR

## 文档更新频率

| 文档类型 | 更新时机 | 健康阈值 |
|----------|----------|----------|
| API 文档 | API 变更时（实时） | >7 天未更新警告 |
| 数据库设计 | Schema 变更时（实时） | >7 天未更新警告 |
| 变更日志 | 提交时（实时） | >7 天未更新警告 |
| 测试用例 | 代码实现后（实时） | >14 天未更新警告 |
| 缺陷记录 | 发现/修复时（实时） | 按需 |
| 架构设计 | 架构变更时（按需） | >30 天未更新警告 |
| 模块/详细设计 | 设计变更时（按需） | >30 天未更新警告 |
| 业务需求 | 需求确认后（按需） | 按需 |
| 系统需求 | 需求分析后（按需） | 按需 |

## 文件命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 业务需求 | `BR-{序号}-{主题}.md` | `BR-001-user-login.md` |
| 系统需求 | `SR-{序号}-{主题}.md` | `SR-001-user-login.md` |
| API 文档 | `{功能}-api.md` | `login-api.md` |
| 数据库设计 | `{表名}-design.md` | `users-design.md` |
| 模块设计 | `{模块}-design.md` | `auth-design.md` |
| 详细设计 | `{功能}-detail.md` | `login-detail.md` |
| 测试用例 | `{功能}-test.md` | `login-test.md` |
| 缺陷记录 | `DEFECT-{序号}-{主题}.md` | `DEFECT-001-login-crash.md` |
| 变更日志 | `v{版本}.md` | `v1.2.0.md` |

## 提交前检查清单

- [ ] 文档格式符合标准模板
- [ ] 文档内容完整，无空占位符（`{{xxx}}`）
- [ ] 文档与代码一致
- [ ] 变更日志已更新
- [ ] 测试用例已更新
- [ ] 所有链接有效
- [ ] 时间戳已更新

---

## 自动化要求

### 优先自动化的文档

| 文档 | 自动化方式 |
|------|------------|
| API 文档 | 从路由定义自动生成 |
| 数据库设计 | 从 Schema 文件生成 |
| 变更日志 | 从 Git 提交生成 |

### 半自动化的文档

| 文档 | 说明 |
|------|------|
| 架构概览 | AI 生成初稿 + 人工完善 |
| 模块设计 | AI 生成初稿 + 人工审核 |

---

**记住**: 文档与代码同等重要，保持同步是团队协作的基础。
